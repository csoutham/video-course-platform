# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch state.

## Reporting a Vulnerability

Please report vulnerabilities privately by emailing: `chris@cjsoutham.com`.

Include:

- Affected component and version/commit
- Reproduction steps or proof of concept
- Expected vs actual behaviour
- Impact assessment

Do not create public issues for unpatched security vulnerabilities.

## Response Process

1. Acknowledge report within 3 business days.
2. Triage and validate impact.
3. Develop and test fix.
4. Publish fix and disclosure guidance.
